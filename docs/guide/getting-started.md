# Updating This Site

Use this page as the lightweight checklist for keeping the documentation useful
between meetings.

## Update Flow

- Start from the [overview](../index.md) and decide which analysis section changed.
- Update only the page that owns the presented material.
- Add meeting outcomes to [recents](../meetings/recents.md).
- Keep personal debugging notes out of this site.

## Content Rules

| Rule | Why it matters |
| --- | --- |
| Use stable plot filenames | Keeps links from changing between meetings |
| Add short plot captions | Makes slides easier to reconstruct later |
| Record decisions separately from status | Prevents meeting notes from becoming a changelog |
| Keep campaign/year labels explicit | Avoids mixing Run-2 and Run-3 assumptions |
